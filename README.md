# DreamLog 🌙

**"Five stars for your dreams."**

A manual sleep diary — no wearables, no sensors, no accounts. Log bedtime, wake
time, and a 5-emoji quality rating in two taps, then watch clean cosmic-purple
charts track your trends, averages, and streaks.

This repo contains three deliverables:

| Folder            | What it is                                             |
| ----------------- | ------------------------------------------------------ |
| `backend/`        | FastAPI + async SQLAlchemy API (SQLite dev, Postgres-ready) |
| `mobile/`         | Expo SDK 54 React Native app (expo-router, Reanimated, SVG charts) |
| `privacy-policy/` | Static privacy policy page (Google Play requirement)   |

---

## 1. Backend — FastAPI

### Requirements
- Python 3.11+ (tested on 3.11 and 3.14)

### Run locally
```bash
cd backend
python -m venv .venv
# Windows:  .venv\Scripts\activate     (or use .venv/Scripts/python.exe directly)
# macOS/Linux:  source .venv/bin/activate
pip install -r requirements.txt

# Apply migrations (creates dreamlog.db). The app also auto-creates tables on
# startup for convenience, but Alembic is the source of truth.
alembic upgrade head

# Start the API (http://localhost:8000, docs at /docs)
uvicorn app.main:app --reload
```

The interactive OpenAPI docs live at **http://localhost:8000/docs**.

### Environment variables
Copy `backend/.env.example` to `backend/.env` and adjust:

| Var            | Default                                | Notes                              |
| -------------- | -------------------------------------- | ---------------------------------- |
| `DATABASE_URL` | `sqlite+aiosqlite:///./dreamlog.db`    | Async URL. Use `postgresql+asyncpg://…` for Postgres |
| `CORS_ORIGINS` | `*`                                    | Comma-separated allowed origins    |

For Postgres also install the driver:
```bash
pip install -r requirements-postgres.txt
```

### Tests
```bash
cd backend
pytest            # 15 tests: scoring, stats/streaks, upsert, 400/404 handling
```

### Auth model
Every request carries an `X-Device-Id` header (a UUID generated by the app on
first launch). The backend implicitly registers a `Device` row the first time it
sees a new UUID — there are no passwords or sessions. A malformed UUID returns
`400`; a missing entry returns `404`.

### Key endpoints (`/api/v1`)
- `POST /devices/register`
- `GET/PUT/DELETE /entries/{date}`, `GET /entries?range=7d|30d|90d|all`
- `GET /stats/summary?range=…` — averages, trends, streaks, distribution, series
- `GET/PUT /settings`
- `GET /export`, `DELETE /data`

## 2. Mobile app — Expo SDK 54

### Requirements
- Node 18+ and npm
- The Expo Go app (SDK 54) or an Android emulator / iOS simulator

### Install & run
```bash
cd mobile
npm install
# Align native module versions to the exact SDK 54 set (recommended once):
npx expo install --fix
npx expo start
```

Then press `a` (Android) or `i` (iOS), or scan the QR code with Expo Go.

> `package.json` pins the intended packages, but `npx expo install --fix`
> reconciles any native-module minor versions with SDK 54.0.8 so the dev build
> is guaranteed compatible.

> The app assets (icon, adaptive icon, splash) are generated by
> `python scripts/gen_assets.py` and are already committed. Re-run it if you
> want to regenerate them.

### Pointing the app at your backend
The API base URL is read from the `API_URL` env var (falls back to the Android
emulator loopback `http://10.0.2.2:8000/api/v1`):

```bash
# Physical device on the same network — use your machine's LAN IP:
API_URL=http://192.168.1.20:8000/api/v1 npx expo start

# iOS simulator can use localhost:
API_URL=http://localhost:8000/api/v1 npx expo start
```

### App structure
```
mobile/
  app/                      # expo-router routes
    _layout.tsx             # fonts, onboarding gate, store hydration, LogSheet host
    onboarding.tsx          # 3-slide intro → creates device UUID
    (tabs)/                 # Today · History · Insights · Settings
  src/
    api/                    # device identity, axios client, typed endpoints
    store/                  # Zustand: entries (offline cache + write queue), settings, ui
    components/             # ArcGauge, charts, BottomSheet, LogSheet, FloatingTabBar…
    lib/                    # score (mirrors backend), date, csv, exporter, quality
    notifications/          # local expo-notifications reminders
    theme/                  # color tokens, typography, spacing
```

### Notable behaviors
- **Offline-first reads:** last-fetched entries are cached to AsyncStorage, so
  the app is never blank without a network.
- **Write queue:** failed saves are queued and retried; the UI shows a
  "couldn't save — will retry" state rather than failing silently.
- **Optimistic UI:** a logged night appears instantly, then reconciles with the
  server. The sleep-score formula in `src/lib/score.ts` is an exact mirror of
  `backend/app/services/scoring.py` so optimistic scores match the server.
- **Contextual permissions:** notification permission is only requested when you
  enable a reminder in Settings.

## 3. Producing a release AAB (Google Play)

Uses [EAS Build](https://docs.expo.dev/build/introduction/). Config is in
`mobile/eas.json`.

```bash
cd mobile
npm install -g eas-cli
eas login
eas build:configure          # first time only
eas build -p android --profile production   # → signed .aab
```

The `production` profile builds an `app-bundle` (AAB) and auto-increments the
version code. Package name: `com.dreamlog.app`.

### Play Store checklist covered here
- Adaptive icon + dark splash matching the theme (`assets/`, `app.json`).
- Working privacy policy page in `privacy-policy/index.html` — host it and set
  its URL in `app.json → extra.privacyPolicyUrl` (and the Play console).
- No crashes on cold start with zero data (designed empty states everywhere).
- Notifications permission requested contextually, not on launch.

## 4. Privacy policy

`privacy-policy/index.html` is a self-contained static page. Host it anywhere
(GitHub Pages, Netlify, your own domain) and set the resulting URL in
`mobile/app.json → extra.privacyPolicyUrl`. It accurately states that the app
stores only an anonymous device UUID and collects no personal data.
